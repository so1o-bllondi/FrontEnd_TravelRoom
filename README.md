# TravelRoom — Планирование путешествий вместе

Веб-приложение для совместного планирования путешествий. Создано в рамках курсовой работы по дисциплине «Фронтенд-разработка».

## Описание проекта

TravelRoom — это современное веб-приложение для планирования путешествий в группе. Пользователи могут:
- Создавать поездки с указанием города, дат и приглашением участников по email
- Использовать готовые маршруты из популярных направлений
- Обсуждать планы в чате с участниками
- Добавлять места для посещения через Яндекс Карты
- Голосовать за предложенные места
- Формировать итоговый маршрут поездки
- Получать прогноз погоды и информацию о городе

## Основные функции

✈️ **Создание поездок** — город, страна, даты, участники по email  
🗺️ **Готовые маршруты** — шаблоны поездок с уже подготовленными местами  
🌍 **Яндекс Карты** — поиск и отметка мест на интерактивной карте  
💬 **Чат поездки** — обсуждение планов в реальном времени  
📍 **Добавление мест** — через поиск на карте или вручную  
👍 **Голосование** — за и против предложенных мест  
🛤️ **Итоговый маршрут** — финальный план с выбранными местами  
☁️ **Прогноз погоды** — реальные данные через Open-Meteo API  
📧 **Email-приглашения** — уведомления участникам через EmailJS  
🌐 **Информация о городе** — данные из Wikipedia API  
🔔 **Browser notifications** — локальные уведомления в браузере

## Технологии

- **React** — библиотека для создания интерфейса
- **Vite** — сборщик и dev-сервер
- **React Router** — маршрутизация
- **Яндекс Карты JavaScript API 2.1** — интерактивная карта
- **Open-Meteo API** — прогноз погоды (не требует ключа)
- **Wikipedia/Wikimedia REST API** — информация о городах (не требует ключа)
- **EmailJS** — отправка email-уведомлений (опционально)
- **Browser Notifications API** — уведомления в браузере
- **lucide-react** — иконки
- **localStorage** — хранение данных

## Установка и запуск

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```bash
cp .env.example .env
```

Откройте `.env` и укажите необходимые ключи:

```env
# Яндекс Карты (обязательно для работы карты)
VITE_YANDEX_MAPS_API_KEY=ваш_ключ_яндекс_карт
VITE_YANDEX_SUGGEST_API_KEY=ваш_ключ_suggest_api

# EmailJS (опционально, для email-уведомлений)
VITE_EMAILJS_SERVICE_ID=ваш_service_id
VITE_EMAILJS_INVITE_TEMPLATE_ID=ваш_template_id_приглашений
VITE_EMAILJS_MESSAGE_TEMPLATE_ID=ваш_template_id_сообщений
VITE_EMAILJS_PUBLIC_KEY=ваш_public_key

# OpenTripMap (опционально)
VITE_OPENTRIPMAP_API_KEY=ваш_opentripmap_key
```

#### Получение ключа Яндекс Карт

1. Перейдите на [developer.tech.yandex.ru](https://developer.tech.yandex.ru/)
2. Зарегистрируйтесь или войдите
3. Перейдите в раздел «JavaScript API и HTTP Геокодер»
4. Нажмите «Получить ключ» и создайте новый ключ
5. Скопируйте ключ в `.env` как `VITE_YANDEX_MAPS_API_KEY`

**Важно:** Без ключа Яндекс Карт приложение будет работать, но вместо карты будет показано сообщение о необходимости настройки.

#### Настройка EmailJS (опционально)

EmailJS используется для отправки email-приглашений участникам и уведомлений о сообщениях в чате.

1. Зарегистрируйтесь на [emailjs.com](https://www.emailjs.com/)
2. Создайте Email Service (например, Gmail)
3. Создайте два Email Template:
   - **Шаблон приглашения** с параметрами: `recipient_email`, `trip_title`, `trip_city`, `trip_country`, `trip_dates`, `organizer_name`, `app_name`
   - **Шаблон сообщения** с параметрами: `recipient_email`, `trip_title`, `trip_city`, `sender_name`, `message_text`, `app_name`
4. Скопируйте Service ID, Template IDs и Public Key в `.env`

**Важно:** Без настройки EmailJS приложение будет работать, но email-уведомления будут недоступны. Приложение покажет статус «EmailJS не настроен».

### 3. Запуск dev-сервера

```bash
npm run dev
```

Приложение откроется автоматически в браузере по адресу [http://localhost:5173](http://localhost:5173)

Альтернативный способ:

```bash
npm start
```

### 4. Запуск через VS Code

В проекте настроены tasks и launch конфигурации для VS Code:

- **Run Task** → `npm: dev` — запустить dev-сервер
- **Run and Debug** → `Launch Chrome against localhost` — запустить с отладкой

### 5. Сборка проекта

```bash
npm run build
```

Результат сборки будет в папке `dist/`

### 6. Просмотр production-сборки

```bash
npm run preview
```

## Использование

### Создание поездки вручную

1. Перейдите на страницу «Создать поездку»
2. Заполните форму:
   - Название поездки
   - Страна и город
   - Даты начала и окончания
   - Email участников (через запятую)
3. Нажмите «Создать поездку»

При создании поездки автоматически определяются координаты города через Open-Meteo Geocoding API. Если настроен EmailJS, участникам отправляются email-приглашения.

### Создание поездки из готового маршрута

1. На главной странице выберите один из популярных маршрутов
2. В модальном окне просмотрите список мест маршрута
3. Введите email участников
4. Нажмите «Создать поездку по маршруту»

Поездка будет создана со всеми местами и координатами уже добавленными на карту.

### Работа с поездкой

На странице поездки доступны:

- **Чат** — обсуждение с участниками
- **Яндекс Карта** — просмотр мест и поиск новых
- **Погода** — прогноз на даты поездки из Open-Meteo
- **Информация о городе** — данные из Wikipedia
- **Добавление мест** — через поиск на карте или вручную
- **Голосование** — за и против мест
- **Формирование маршрута** — добавление мест в итоговый план

### Поиск и добавление места

**Способ 1: Поиск на карте**
- Введите название места в поле поиска (например: «Эйфелева башня»)
- Нажмите Enter или кнопку поиска
- После нахождения появится карточка найденного места
- Выберите категорию и добавьте описание
- Нажмите «Добавить в поездку»

**Способ 2: Клик по карте**
- Кликните на карте в нужной точке
- Появится временная метка и карточка
- Введите название, категорию и описание
- Нажмите «Добавить в поездку»

**Способ 3: Ручной ввод координат**
- Прокрутите страницу до формы добавления места
- Заполните название, категорию, описание
- Введите широту и долготу
- Нажмите «Добавить место»

### Погода

Прогноз погоды загружается автоматически при открытии поездки. Используется бесплатный Open-Meteo API.

**Важно:** Open-Meteo предоставляет прогноз на ближайшие 16 дней. Если даты поездки выходят за этот диапазон, приложение покажет ближайший доступный прогноз с соответствующим предупреждением.

### Информация о городе

Данные о городе загружаются из Wikipedia/Wikimedia REST API. Приложение пытается найти русскую версию статьи, если не найдена — использует английскую, если и её нет — показывает fallback-описание.

### Email-уведомления

**Приглашения:** При создании поездки с участниками автоматически отправляются email-приглашения (если настроен EmailJS).

**Сообщения в чате:** В настройках уведомлений можно включить отправку email при новых сообщениях. Письма отправляются всем участникам, кроме автора сообщения.

## API-ключи и безопасность

- **Яндекс Карты:** Ключ хранится в `.env` и не попадает в репозиторий благодаря `.gitignore`
- **Open-Meteo:** Не требует ключа, бесплатный для некоммерческого использования
- **Wikipedia API:** Не требует ключа, использует публичный REST API
- **EmailJS:** Ключи хранятся в `.env`

**Важно:** Не публикуйте файл `.env` в публичных репозиториях.

## Ограничения прототипа

- Нет backend-части — данные хранятся только в localStorage
- Демонстрационная авторизация без реальной аутентификации
- Email-уведомления требуют настройки EmailJS
- Прогноз погоды ограничен 16 днями от текущей даты
- Координаты городов определяются через Open-Meteo Geocoding API
- Для работы карты необходим API-ключ Яндекс Карт
- Готовые маршруты используют статические данные из `routePresets.js`
